3-1 프로젝트 주제: 안심복약 AI (Medi-Check)
프로젝트 소개
"먹고 있는 약들, 과연 같이 먹어도 안전할까요?"

고령화 사회로 접어들며 만성질환자의 복용 약물 개수가 늘어나고, 영양제 섭취 또한 보편화되었습니다. 하지만 약물 간의 상호작용(궁합)을 개인이 파악하기는 매우 어렵습니다. **'안심복약 AI'**는 카메라 스캔 한 번으로 복잡한 약 이름을 자동 등록하고, 국가 공인 데이터를 바탕으로 약물 간 충돌 위험을 실시간 분석하여 안전한 복약 환경을 제공하는 헬스케어 솔루션입니다.

핵심 기능
📸 약 봉투/처방전 OCR 자동 등록

사용자가 약 봉투나 처방전을 촬영하면, AI(GPT-4o)가 복잡한 텍스트 속에서 실제 **'의약품 명칭'**만 정확히 추출하여 리스트업합니다.

수동 입력의 번거로움을 없애고, 약 이름을 몰라 등록을 포기하는 진입장벽을 제거합니다.

⚠️ 약물 상호작용(DUR) 실시간 분석

사용자가 새로 등록한 약이 기존에 복용 중인 약이나 영양제와 함께 먹었을 때 위험(병용 금기, 성분 중복 등)이 있는지 약학정보원 및 식약처 공공 데이터와 대조하여 즉시 경고합니다.

"현재 복용 중인 A약과 새로 등록한 B약을 함께 드시면 혈압 저하 위험이 있습니다!"와 같은 구체적인 주의사항을 제공합니다.

🗓️ 지능형 복용 타임라인 & 스마트 알림

각 약의 복용 지침(식전/식후, 아침/점심/저녁)을 분석하여 최적화된 **'통합 복약 스케줄'**을 자동으로 생성합니다.

단순 알람이 아니라, 약물 특성에 맞춰 "지금 식사 직후이니 A약을 드실 시간입니다"와 같은 맥락 기반 푸시 알림을 발송합니다.

기술 스택
Frontend
React Native (Expo): iOS 및 Android 크로스 플랫폼 앱 개발

💡 선정 이유: 카메라 API 제어가 직관적이며, 건강 관리 앱 특성상 실시간 푸시 알림(Expo Notifications) 구현이 용이합니다. 또한 'Expo Go'를 통해 교수님께 실시간 시연(Demo)을 보여드리기 최적입니다.

Backend & Database
FastAPI (Python): 빠르고 가벼운 비동기 서버 구축

💡 선정 이유: 공공 데이터 API(XML/JSON)를 호출하고 가공하는 데 파이프라인 처리가 매우 빠르며, AI 모델 연동이 매끄럽습니다.

PostgreSQL: 관계형 데이터베이스

💡 선정 이유: 약물 성분 간의 복잡한 상관관계와 사용자별 복용 스케줄 데이터를 체계적으로 관리하기 위해 선정하였습니다.

AI & Data Processing
Google Cloud Vision API: 사진 속 텍스트 추출 (OCR 핵심 엔진)

OpenAI API (gpt-4o): 추출된 텍스트에서 약 이름 정제 및 사용자 친화적인 복용 주의사항 요약 제공

공공데이터포털 (DUR API): 식약처 제공 병용 금기 및 임부 금기 등 의약품 안전성 정보 연동

💡 기대 효과
약물 사고 예방: 병용 금기 약물 섭취로 인한 부작용을 사전에 차단하여 시니어 및 만성질환자의 안전 보장

복약 순응도 향상: 복잡한 복용 시간을 하나의 타임라인으로 관리하여 약을 거르는 횟수 최소화

디지털 헬스케어 경험: 어렵게 느껴지는 의약 전문 정보를 AI를 통해 일반인이 이해하기 쉬운 언어로 전달
